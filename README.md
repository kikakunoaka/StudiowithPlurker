# 噗浪周邊客製工作室｜評價目錄

純前端（HTML／CSS／JS，無需後端、無需 API Key）的廠商評價網站，
資料即時讀取自 Google 試算表：

https://docs.google.com/spreadsheets/d/1fpoLejfAJz6bQwAPA4xwTzQzW21Y-5ABPT1f-vaGwsQ/

## 網站結構

- `index.html` — 主頁
  - 最上方為可折疊的「更新日誌」，內容讀取「更新日誌」分頁
  - 篩選列僅比對「工作室列表」分頁中的 C（營運狀況）、F（委託方式）、
    G（工廠）、H（客製項目）欄
  - 列表僅讀取「工作室列表」分頁，資料從 A2（第一位工作室）開始
  - 每張卡片顯示該列 ICON 欄位的圖片
- `studio.html` — 各工作室分頁模板，透過網址 `?name=工作室名稱`
  讀取「與工作室同名」的分頁：
  - 廠商介紹：讀取 A1:G15，A 欄為欄位標題、B～G 欄為內容，
    每個標題底下的內容以橢圓 TAG 呈現
  - 體驗者感想：從第 18 列開始（第 18 列為標題列，19 列起為資料）
- `js/config.js` — 設定檔（試算表 ID、分頁名稱、篩選欄位、讀取範圍）
- `js/sheets.js` — 讀取 Google 試算表的共用函式
- `js/main.js` / `js/studio.js` — 各頁面邏輯
- `css/style.css` — 樣式

## 第一步：確認 Google 試算表分享設定

共用 →「知道連結的任何人」→ 權限「檢視者」。

## 第二步：上傳到 GitHub 並開啟 GitHub Pages

1. 在 GitHub 建立 repository，把資料夾內所有檔案上傳（保留 `css/`、`js/` 資料夾結構）。
2. Settings → Pages → Source 選 `Deploy from a branch`，Branch 選你的預設分支、
   資料夾選 `/ (root)`，Save。
3. 等 1～2 分鐘取得網址，例如 `https://你的帳號.github.io/repo名稱/`。

## 第三步：日後新增工作室怎麼做？

1. 新增一個分頁，**分頁名稱需與工作室名稱完全一致**。
2. 在該分頁填寫：
   - A1:A15 填寫欄位標題（例如：平台、委託方式、工廠、客製項目…）
   - 對應列的 B～G 欄填入內容（同一標題可有多個值，會各自變成一個 TAG）
   - 第 18 列填寫體驗回報的標題（例如：日期、暱稱、心得、評分），
     第 19 列起逐筆填入體驗者感想
3. 在「工作室列表」分頁新增一列：填入名稱、ICON 圖片網址，
   以及營運狀況／委託方式／工廠／客製項目等分類，會自動出現在主頁與篩選列。
4. 不需要改任何程式碼、不需要重新部署，重新整理網站即可看到新工作室。

## 更新日誌分頁格式

「更新日誌」分頁建議格式：第一欄為日期，其餘欄位為更新內容說明，
每一列會顯示為一筆日誌紀錄。

## 欄位對應與客製

若試算表分頁名稱、欄位標題文字或讀取範圍與此文件假設的不同，
請至 `js/config.js` 修改對應設定值（`listSheetName`、`changelogSheetName`、
`iconColumnName`、`filterColumns`、`profileRange`、`reviewStartRow`）。

## 本機預覽

`fetch()` 讀取外部資料在 `file://` 協定下可能被瀏覽器 CORS 政策擋下，
本機測試建議：

```bash
python3 -m http.server 8000
```

再開啟 `http://localhost:8000`。上傳到 GitHub Pages 後就沒有這個限制。

## 已知限制

- Google 試算表的公開查詢端點偶爾會有幾秒快取延遲，屬正常現象。
- 若某工作室分頁尚未填寫 A1:G15 或第 18 列以後的內容，對應區塊會顯示
  「尚未填寫」或「目前還沒有體驗心得」等提示文字，不會顯示錯誤。
